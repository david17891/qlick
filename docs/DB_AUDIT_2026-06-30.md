## Lecciones aprendidas para futuras sesiones Mavis

1. **`CREATE POLICY` no es idempotente en Postgres.** Migraciones que agregan
   policies requieren `DROP POLICY IF EXISTS` antes o aceptar que la migration
   falle si la policy ya existe. Auditoria previa antes de marcar como `applied`.
2. **El comando `Set-Clipboard` puede ser silencioso.** Si el contenido parece
   haberse pegado pero el Run no tuvo efecto, usar `npx supabase db execute --file`.
3. **El SQL Editor del dashboard tiene limite de caracteres.** Para migrations
   largas usar el CLI (`npx supabase db push`) en lugar del editor.
4. **`migration repair --status applied` es traicionero.** Marca en el ledger
   sin tocar el schema real. Usar SOLO cuando estes 100% seguro de que el
   efecto ya esta aplicado.
5. **`.env.local` con bytes no-ASCII rompe el CLI de Supabase.** Limpiar
   comentarios con mojibake antes de correr `db push`. Usar `clean_env_comments.py`
   del scratchpad.
6. **PowerShell 5.1 no acepta `$env:$name = ...` para env vars dinamicas.**
   Usar `Set-Item -Path "Env:$name" -Value $value -Force` en su lugar.
